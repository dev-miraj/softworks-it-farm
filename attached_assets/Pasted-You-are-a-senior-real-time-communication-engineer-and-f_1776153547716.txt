You are a senior real-time communication engineer and full-stack architect.

Your task is to build a **FULLY CUSTOM, INTERNET-BASED AI CALLING SYSTEM** (like WhatsApp/Messenger calls) using WebRTC.

⚠️ IMPORTANT:

* NO third-party calling APIs (Twilio, etc.)
* 100% self-hosted system
* Full control over voice, call flow, and logic

---

# 🎯 CORE GOAL

Build a system where:

1. Admin creates a call session
2. A unique call link is generated (/call/:token)
3. User opens the link → connects via WebRTC
4. AI voice speaks (custom or TTS)
5. User can:

   * Press keypad buttons (DTMF-style UI)
   * OR speak (future-ready)
6. System processes input → triggers actions
7. Result saved + webhook triggered

---

# 🧠 SYSTEM ARCHITECTURE

Frontend:

* React + TypeScript
* WebRTC client
* Call UI with keypad

Backend:

* Node.js (Express)
* WebSocket signaling server (Socket.IO)
* Call session management

Media Layer:

* WebRTC (peer-to-peer)
* STUN/TURN server (Coturn self-hosted)
